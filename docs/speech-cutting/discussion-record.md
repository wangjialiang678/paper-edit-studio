# 语音切分算法讨论记录

## 初始动机

项目已经能通过 FunASR 得到词级时间戳，但直接按时间戳剪切时，偶尔会剪掉前一个词的尾音或后一个词的字头。目标是把“安全切点”先做成可独立调研和测试的算法，不先扩展整套视频剪辑工具。

## 最初假设

最初提出的直觉包括：

- 每个人是否有特定频率或特定波形；
- 是否能从混合音频中绘制某个人的专属波形；
- 是否能把波形当成图像，用视觉算法找间隔；
- 如果能把目标人从背景音乐、噪声和其他说话人中提取出来，是否就能找到他的句间或词间停顿。

调研后的修正是：人没有固定独占频段，声纹是跨多帧语音形成的统计特征。把频谱交给神经网络处理本身就是现代音频算法，但把整条波形截图后交给通用视觉模型不会增加毫秒级信息。

## 形成的技术拆分

| 子问题 | 对应技术 | 对切点的作用 |
|---|---|---|
| 这里有没有人声 | VAD/SAD | 找可能的静音区 |
| 谁在什么时候说 | diarization | 避免剪进下一个人的话头 |
| 这是不是目标人 | speaker embedding / verification | 将匿名说话人映射到目标人 |
| 目标人此刻是否在说 | TS-VAD | 给出目标人停止/开始的逐帧概率 |
| 只保留目标人的分析轨 | Target Speaker Extraction | 在噪声、音乐或重叠语音中提供辅助证据 |
| 已知文字在哪里 | CTC/MFA/强制对齐 | 估计字词声学范围 |
| 最终在哪里剪 | 安全区间融合 | 先防吞字和残词，再优化自然度 |

## video-use 审查

审查了本地下载的 `browser-use/video-use`。其主要流程是“ElevenLabs 转写 → Agent 生成 EDL → 渲染 → 切点自评估 → 必要时重渲染”。可复用的是 EDL、渲染、时间线可视化和闭环复查思路；仓库本身仍直接使用转写时间戳，没有证明能解决中文吃字问题。

## 对齐基准实验

建立了 24 个真实中文边界，对比 FunASR、CTC、MFA 和 ElevenLabs Forced Alignment 适配器。用户无法为 24 个样本人工填写毫秒级可接受区间，因此评估方式改为直接生成多种方案的成品盲听。

## 第一轮盲听偏差

第一轮主要删除静音、语气词或短停顿。用户听后指出，成品像完整录音，听不出明显发生了剪切。这不是用户没有听仔细，而是实验任务没有让“发生跳剪”这件事变得可感知。

因此实验任务被重新定义为：九句话保留第 1、3、5、7、9 句，完整删除第 2、4、6、8 句，并为每个切口生成三秒放大片段。

## 整句跳剪的最终反馈

四个匿名版本对应：

- A：`hybrid_safe`
- B：`ctc_char`
- C：`funasr_direct`
- D：`mfa_word`

四个版本都通过了自动质检，用户完成听感比较后表示“这四个也听不出区别，好像都差不多”。这只能将该样本记为“未观察到可辨别的主观差异”，不足以宣布四者并列最优或已通过泛化性验收。

## 对“声纹滤波”的最终回答

这个设想对应真实存在的 Target Speaker Extraction 和 TS-VAD：先用一段目标人录音生成声纹特征，再让模型动态判断混合音频中哪些时频成分属于目标人。对剪辑而言，最有价值的不一定是“干净目标人音轨”，而是“目标人逐帧活动概率”。

该路线能帮助句间停顿、多人轮流和背景音乐场景，但不能单独解决没有静音的词间连读，因此必须与强制对齐和安全区间搜索组合。

## 当前共识

- 不用更多普通句间素材反复盲听。
- 不因为模型时间戳彼此不同，就把“分歧更大”当成“更准”。
- 不把声纹、VAD、diarization 或强制对齐中任何一个模块当成单独真相源。
- 未来只收集产品中真正听得见的失败切点，用失败样本驱动后续模型选型。
