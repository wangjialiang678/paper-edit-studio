# 语音切分算法后续路线

## 原则

不再为了“让模型数量更多”而增加模型。只有当真实产品出现可听见的吞字、残词、串人或接缝突变时，才将该片段收入失败集，然后用它驱动方案选型。

## P0：建立真实失败集

每个失败切点保存：

- 原视频/音频片段和时间坐标；
- 保留文字、删除文字和 FunASR 锚点；
- 失败类型：吞字、残词、拖音、呼吸、爆音、BGM 突变、串入他人、overlap；
- 一个可接受切分区间，或明确标记“无安全点”；
- 原算法的切点、置信度和回退原因。

优先收集五类难例：无停顿快速连读、词尾拖音、持续 BGM、噪声/混响、两人重叠或抢话。

## P1：重新评估当前混合方案

在真实失败集上对比：

1. FunASR 直接时间戳；
2. FunASR + 局部 CTC 对齐；
3. FunASR + 独立 VAD + 局部声学候选；
4. 当前 `hybrid_valley`；
5. 无安全点时的拒绝切分机制。

先比较“吞字率、残词率、禁区命中率”，再比较紧凑度，不使用单一平均毫秒误差代替听感质量。

## P2：多人和背景声疑难回退

只在 P0 失败集证明必要时实现：

- diarization 产出说话人时间线和 overlap 禁切区；
- 固定主播场景用 ECAPA/CAM++ 注册目标声纹；
- TS-VAD 产出目标人活动概率；
- 只对低置信度小窗口运行 Target Speaker Extraction；
- 增强/分离结果只用于分析，需校准相对原轨的延迟和漂移。

## P3：尝试更强对齐或学习式排序

只在失败集足够大时考虑：

- 测试 ElevenLabs Forced Alignment，但必须使用同一批中文失败样本和同一人工标注；
- 替换当前非商业 CTC 研究权重，重新核验许可证；
- 用真实成功/失败切点训练小型候选排序器；
- 保留硬护栏和拒绝切分，不让学习模型直接越过语义安全区。

## 暂不做

- 不用固定频段滤波器当作声纹分离。
- 不把波形截图交给通用视觉模型找毫秒级词界。
- 不全量运行重型分离模型。
- 不在没有人工标注时宣布某个对齐模型“更准”。
- 不再用四个普通句间版本迫使用户进行无差别盲听。
