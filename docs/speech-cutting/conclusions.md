# 语音切分算法当前结论

## 要解决的真实问题

输入是已知要保留和删除的文字片段，输出不应只是一个“最精确毫秒”，而应是：

- 可以安全剪切的时间区间；
- 该区间内最自然的候选点；
- 对保留语音、被删语音、下一说话人和重叠语音的安全余量；
- 没有安全点时，明确选择保守留白、合并片段或拒绝切分。

“吃掉一个字头”的代价高于“多留几十毫秒气口”，因此算法的损失函数应当是不对称的。

## 已验证结论

1. **ASR 词时间戳是粗锚点，不是最终切点。** FunASR 适合转写、语义选段和快速锚定，边界附近仍需局部校准和安全余量。
2. **词与词之间不一定存在客观静音点。** 连读、轻声和协同发音会让语音跨越文字词界，有些边界只能表示为“可接受区间”或“不可安全切分”。
3. **声纹不是某个人独占的固定频段。** 现代声纹系统用数秒语音形成高维 `speaker embedding`，而不是用一个带通滤波器分离说话人。
4. **声纹条件的动态“滤波”确实存在。** Target Speaker Extraction、VoiceFilter、SpeakerBeam 会根据目标声纹，对每个时间—频率区块动态估计属于目标人的概率。它更像不断变化的智能滤波器，不是固定频段。
5. **对剪辑更直接的产物是目标说话人活动概率。** TS-VAD 逐帧回答“目标人此刻是否在说话”，可以在背景音乐或其他人发言时给出目标人的停顿证据。
6. **声纹不能单独给出词边界。** 声纹回答“是谁”，VAD/TS-VAD 回答“是否在说”，diarization 回答“谁在什么时间说”，CTC/MFA 等强制对齐才直接估计字词位置。
7. **背景音乐下，混合轨 RMS 不能作为主证据。** RMS 只看总能量，无法判断能量来自目标人、其他人还是音乐。增强或分离音轨只应用于分析，最终剪辑应作用于原始媒体。
8. **多人重叠语音默认是禁切区。** 目标说话人提取可作为疑难片段的局部回退，但相似声线、混响和目标人缺席会导致串音或误抑制。

## 现有实验能够证明什么

- 24 个中文边界上，FunASR 和 CTC 都能输出 24/24，MFA 输出 19/24；由于没有人工可接受区间，这些数据只能评估覆盖率和模型分歧，不能排准确率名次。
- ElevenLabs Forced Alignment 接口已实现和测试，但本地没有 API Key，24/24 都是 `unavailable`，因此没有证据说它比 FunASR、CTC 或 MFA 更适合中文视频。
- 整句跳剪盲测中，FunASR、CTC、MFA 和混合方案都完整删除了四句话并通过自动质检。用户听完 A/B/C/D 后的最终反馈是“四个都听不出明显区别”。
- 这说明普通句间停顿样本对四种方案都不构成挑战，不说明某个方案被听感证明更好。

## 当前工程决策

```text
FunASR 文本和粗时间戳
        ↓
边界附近局部对齐 / token 护栏
        ↓
独立 VAD、说话人和 overlap 禁切证据
        ↓
原始/增强分析轨的局部声学候选
        ↓
硬约束过滤 → 自然度排序 → 保守回退
        ↓
在原始媒体上导出
```

- Studio 现在保留 `hybrid_valley` 作为默认，理由是现有代码成熟度和保守设计。整句盲听中的混合策略是另一个 `hybrid_safe`，不是 `hybrid_valley`，因此该盲听不能证明当前 Studio 默认策略更好。
- 暂不增加 ElevenLabs 依赖和 API 成本。
- 暂不把完整语音分离模型放到全量默认链路。
- 将 TS-VAD/目标说话人提取保留为多人、背景音乐和重叠语音的疑难切点回退路线。

## 尚未证明的事

- ElevenLabs 是否在真实中文口语上优于本地对齐方案。
- CTC、MFA、FunASR 或混合方案在“无停顿连读、拖音、背景音乐、噪声、两人重叠”上的真实优劣。
- TS-VAD 或目标说话人提取能否显著降低本项目的吞字率。目前只完成了调研和脚本入口，没有真实模型推理结果。
- 现有样本没有人工 gold，因此不能报告词边界命中率。
